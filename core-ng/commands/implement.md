# 功能实现工作流

基于已完成的规划文档直接实现。核心目标：严格按方案执行，确保实现与规划一致。

## 前置验证

**必须满足**：
- 存在规划文档（`$ARGUMENTS` 或默认 `PLAN.md`）
- 规划文档包含完整的架构方案和任务清单
- 用户已批准方案

---

## 阶段 1：加载规划

**目标**：读取并确认方案内容

**执行步骤**：
1. 读取规划文档（`$ARGUMENTS` 或 `PLAN.md`）
2. 提取关键信息：
   - 功能概述
   - 架构决策和理由
   - 文件变更清单
   - 实现任务及依赖
   - 测试策略
3. 向用户简要确认规划内容
4. **询问用户是否按此方案开始实现**

---

## 阶段 2：实现准备

**目标**：建立实现所需的代码上下文

**执行步骤**：
1. 阅读规划中的关键文件清单
2. 创建任务列表（TodoWrite）：
   - 按规划的任务顺序
   - 标注依赖关系
3. 确认实现顺序（按依赖关系执行）

---

## 阶段 3：代码实现

**目标**：严格按方案实现

**前置条件**：用户已明确批准开始

**执行步骤**：
1. **等待用户明确批准**
2. 按任务列表顺序实现：
   - 阅读相关文件建立上下文
   - 严格遵循规划的文件变更内容
   - 遵循代码库命名和结构约定
   - 每完成一个任务更新 todo 状态
3. 完成所有任务

**实现原则**：
- 不偏离规划方案
- 遇到问题及时反馈，不擅自修改方案
- 代码质量标准：清晰、可测试、符合约定

---

## 阶段 4：测试验证

**目标**：验证功能正确性

**执行步骤**：
1. 启动 1-2 个 `code-tester` 代理：
   - 按规划的测试策略设计用例
   - 编写单元测试和集成测试
   - 执行测试验证结果
2. 整合测试结果：
   - 测试覆盖率
   - 失败测试及原因
   - 待覆盖场景
3. **向用户呈现测试结果**
4. 如有失败，询问处理方式并修复

**代理提示词**：
```
按以下测试策略验证 [功能名]：[规划的测试策略内容]
```

---

## 阶段 5：质量审查

**目标**：全面检查代码质量

**执行步骤**：
1. 并行启动 4 个 `code-reviewer` 代理：
   - **代理1**：功能正确性（逻辑错误、边界处理、并发问题）
   - **代理2**：代码质量（DRY、复杂度、可读性）
   - **代理3**：项目约定（命名规范、目录结构、依赖管理）
   - **代理4**：规范校验（Checkstyle执行）
2. 整合所有审查发现
3. 按严重性排序问题
4. **向用户呈现发现**：
   - 关键问题（置信度 90+）
   - 重要问题（置信度 80-89）
   - Checkstyle 失败
5. **询问用户处理方式**：
   - 立即修复
   - 稍后修复
   - 按原样继续
6. 根据用户决定处理问题

---

## 阶段 6：总结

**目标**：记录完成工作

**执行步骤**：
1. 标记所有 todo 完成
2. 总结：
   - 构建内容
   - 关键决策回顾
   - 修改的文件列表
   - 测试覆盖率
   - 代码审查结论
   - 后续建议
3. 询问用户是否删除规划文档（可选）

---

## 与规划阶段的关系

```
/plan → 输出 PLAN.md → /implement → 输出 实现代码 + 测试 + 审查报告
```

规划阶段负责分析和设计，实现阶段负责执行和验证。两阶段可由不同人员负责，支持审批流程。
